Impact

When a user is redirected here after login, the browser sends the Referer header containing the previous page URL. Depending on the browser's referrer policy, this can leak:

  • The full URL including secure.backblaze.com origin
  • Session tokens or sensitive parameters if present in the URL
  • Confirmation that the user has a Backblaze account
  • The user just entered valid credentials on the real login page (social engineering value)

Attack scenario:

  1. Attacker sends victim a link: https://secure.backblaze.com/user_signin.htm?redirect_to=@evil.com/phish
  2. Victim sees the real Backblaze login page at secure.backblaze.com
  3. Victim enters valid credentials and logs in
  4. After successful authentication, the app redirects to https://secure.backblaze.com@evil.com/phish
  5. Browser navigates to evil.com/phish â€” this page captures the referrer
  6. Attacker's page could show a fake "session expired" message to phish credentials again
